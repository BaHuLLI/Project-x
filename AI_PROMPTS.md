# AI Prompts

Готовые шаблоны для работы с ИИ в репозитории `Project-x`.

Перед использованием шаблонов ИИ должен читать:

- `AI_WORKFLOW.md`
- `PROJECT_CONTEXT.md`
- `README.md`

## Universal

```text
Работаем в репозитории Project-x.

Сначала прочитай:
- AI_WORKFLOW.md
- PROJECT_CONTEXT.md
- README.md

Правила:
- не работай напрямую в main
- создай отдельную ветку от main
- всегда держи изменения в новой ветке, пока пользователь явно не разрешит пуш или merge в main
- никогда не пушь в main самостоятельно
- никогда не мержи в main самостоятельно
- только пользователь решает, когда изменения можно переносить в main
- делай только минимальные и локальные изменения
- не меняй публичные интерфейсы и архитектуру без отдельного согласования
- не делай массовый рефакторинг без явного запроса
- если меняется процесс работы, инструкции, контекст проекта или способ использования, обнови соответствующий markdown-файл в репозитории

После изменений обязательно запусти:
powershell -ExecutionPolicy Bypass -File .\scripts\local-check.ps1

Если проверки не прошли — сначала исправь проблемы.
Если проверки прошли — покажи, что изменено, и подготовь изменения к merge в main.
```

## UI / UX Change

```text
Работаем в репозитории Project-x.

Сначала прочитай:
- AI_WORKFLOW.md
- PROJECT_CONTEXT.md
- README.md

Задача: доработать UI/UX в WPF-приложении.

Правила:
- работай только в отдельной ветке от main
- не пушь и не мержи в main без явного разрешения пользователя
- сохраняй текущую архитектуру и публичные интерфейсы, если отдельно не согласовано иное
- изменения делай минимальными, локальными и согласованными с существующим стилем проекта
- не превращай задачу по UI в массовый рефакторинг
- учитывай, что продукт должен быть low-friction, быстрым в настройке и понятным для sim-racing пользователя
- если UI-изменение меняет пользовательский сценарий, обнови связанную документацию или контекст

После изменений обязательно запусти:
powershell -ExecutionPolicy Bypass -File .\scripts\local-check.ps1

Если проверки прошли — кратко покажи:
- что изменилось в интерфейсе
- какие экраны/файлы затронуты
- как проверить результат
```
